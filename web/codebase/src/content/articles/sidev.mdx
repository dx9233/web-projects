# ✅ Шаг 1. Установить MSYS2 (рекомендуется)

1. Скачать инсталлер **MSYS2** с официального сайта
2. Установить и запустить **MSYS2 MINGW64** (важно — не MSYS shell).
3. Выполнить обновление:

```bash
pacman -Syu
```

После перезапуска:

```bash
pacman -Syu
```

4. Установить MinGW-w64 (gcc, g++, make):

```bash
pacman -S mingw-w64-x86_64-gcc mingw-w64-x86_64-gdb mingw-w64-x86_64-make
```

---

# ✅ Шаг 2. Добавить MinGW в PATH

В проводнике перейди:

```
C:\msys64\mingw64\bin
```

Скопируй путь → добавь в:

```
Панель управления → Система → Доп. параметры → Переменные среды → Path → Добавить
```

Проверка:

Открой обычный CMD или PowerShell:

```cmd
gcc --version
```

Если показывает версию — работает.

Установи расширения:

✔ C/C++ (Microsoft)
✔ C/C++ Extension Pack *(по желанию)*
✔ Code Runner *(если хочешь запуск кнопкой)*

# ✅ Шаг 4. Создать автоматическую сборку (Ctrl+Shift+B)

1. В VS Code открой проект.
2. Terminal → Configure Build Task → C/C++: gcc.
3. VS Code создаст `.vscode/tasks.json`.

Если не создал — вот готовый вариант:

### **.vscode/tasks.json**

```json
{
  "version": "2.0.0",
  "tasks": [
    {
      "label": "build C",
      "type": "shell",
      "command": "gcc",
      "args": [
        "main.c",
        "-o",
        "main.exe"
      ],
      "group": {
        "kind": "build",
        "isDefault": true
      }
    }
  ]
}
```

Теперь ты можешь нажать **Ctrl+Shift+B** — и VS Code соберёт проект.

# ✅ Шаг 5. Настроить отладку (необязательно)

Создай файл:

### **.vscode/launch.json:**

```json
{
  "version": "0.2.0",
  "configurations": [
    {
      "name": "Debug C Program",
      "type": "cppdbg",
      "request": "launch",
      "program": "${workspaceFolder}/main.exe",
      "cwd": "${workspaceFolder}",
      "MIMode": "gdb",
      "miDebuggerPath": "C:/msys64/mingw64/bin/gdb.exe",
      "externalConsole": false
    }
  ]
}
```
